# 探雷模式（Mine Runner）— 設計規格

> 工作名稱暫定。這是一個**獨立的新玩法模式**，不更動現有的經典對戰。
> 核心想法：角色在地圖上**回合制移動**去找雷，固定斜角 2.5D 視覺。

---

## 1. 定位與技術前提（已定）

- **獨立模式**：與經典對戰並存，不互相干擾。
- **互動模型**：回合制移動（非即時），所以可沿用現有「權威伺服器 + 回合 + 對局事件流」架構。
- **視覺**：固定斜角 2.5D。**先不要用 three.js**，也不養 3D 資產管線。
  - 原型驗手感：直接拿現有 Vue 盤面 `transform: perspective() rotateX()` 斜過來 + 一個會在格間滑動的角色。零新依賴。
  - 手感對了要質感再上 **PixiJS**（等角 sprite，比 three.js 輕一個量級）。
- **可重用**：地雷場生成、鄰格數字、勝負判定（`lib/game.js` 的盤面真相）；房間、重連、觀戰、Replays、對局事件流、紀錄、音效全部照用。只要事件流多記「角色座標」。

---

## 2. 核心迴路（v1，原型範圍）

### 2.1 行動經濟（單一貨幣：行動值＝時間）

- 每次活化（你的回合）把**行動值**補滿到**上限**，在動作間**自由分配**。
- **行動值與時間是同一種貨幣**（決定採「甲」案）：每個動作花掉的行動值 = 它在**時間軸**上推進的距離。行動上限 = 你一次活化能花的時間額度。

### 2.2 回合順序：時間軸（Patchwork / Thebes 式）

- 不是硬性輪流，而是**時間用得最少的一方一直行動**，直到超過對手才換手。
- 效果：一回合做越多 → 時間推越遠 → 對手會**連開好幾次小回合**才輪回你。→ **內建的節奏型後追**（免費的追趕機制）。

### 2.3 三個核心動作

| 動作 | 規則 |
|---|---|
| **移動** | 花行動值逐格走，重新定位 |
| **標記** | **必須在控制範圍（視野）內**才能標；跨回合保留 |
| **揭露** | **不限視野、一次打開**；前提是該格**先前已被你標記**。雷→得分；非雷→公開數字線索 |

- **翻地流程**：一定要 **標記 → 揭露**（兩段式）。
- **標記在地、揭露遙控**：標記要親臨（曝險），揭露可遠端引爆。這正是「讓移動有價值」的關鍵。

### 2.4 偷雷與失效（對稱）

- **對稱規則**：對手要偷，同樣得**親臨標記 → 揭露**（他得先摸進範圍看到你的標記）。
- 先揭露者得雷；**被搶走的格子，另一方的標記/揭露即失效**。
- **失效＝合理情報**：你遙控揭露失敗，本身就告訴你「它已被翻過」。資訊缺口一定會被「行動」補上。

### 2.5 可見性（迷霧）：事實公開、意圖迷霧

- **已翻開的格（數字、被搶的雷）= 全局公開、永久可見。**
- **對手的「標記」（意圖）= 只有在你控制範圍內才看得到**，而且：
  - **離開後標記不變，但你不再收到動態更新** → 你看到的是「上次經過時的快照」。
  - 你只能靠**回去重新看到**、或**揭露失敗**才得知它被偷了。

### 2.6 結果與勝負

- 揭露到**雷** → 得分。
- 揭露到**非雷** → 公開數字線索（同時也把情報免費送給對手 → 這就是亂揭露的天然懲罰，**不另加罰則**）。
- **勝負**：先搶過半的雷（沿用經典規則）。

## 3. 防雪球 / 後追（v1 原型只放兩個）

挑「**假裝不來**」的追趕（綁在真實狀態/位置上，無法靠裝弱得利）：

1. **領先者更曝光（家族 B，重用迷霧）**：搶到的雷越多 → 你的**角色與標記被對手看見的範圍越大**。領先＝樹大招風，更容易被找到、被偷。**零新系統，直接長在迷霧上。**
2. **時間軸節奏追趕**（§2.2，已內建）。

> 通則：**任何「獎勵落後」的補貼型機制都會誘發裝弱**。原型故意只用曝險型與節奏型；補貼型（落後加上限/加 XP）與變數型（晚期雷更值錢、連鎖偷雷獎勵、中央高密度雷區）留作之後不夠平衡時的備案。

## 4. 延後（Phase 2，原型不做）

### 4.1 XP / 等級（先不進原型）

- 數字格上有 **XP，要走過去才拿**；雷的 XP **直接入袋**。
- XP → 升級 → 拉高行動上限。
- **為何延後**：`搶雷→XP→升級→上限變高→搶更多雷` 是**富者愈富的正回饋，會滾雪球**。時間軸能平衡「輪到頻率」但平衡不了「吞吐量」。原型先用**固定行動上限**驗證底層好不好玩，XP 當之後的調味層。
- 屆時的減震方向：上限成長遞減/封頂、升級只買效率不直接加分、落後方 XP 加成。

### 4.2 能力層 / 道具（先不進原型）

- **檢查 / 調查**：不移動就取得情報（看 n×n 確切雷況或數量）。⚠️ 情報類最危險，會讓推理謎題崩塌，要重課金（極少次數/耗整回合/只給數量）。
- **踩 0 → 一次性傳送到任一 edge 格**：把 flood-fill 的 0 翻轉成機動獎勵。
- **道具系統**：上述能力的容器，等核心驗證後一起設計。

## 5. 待拍板的數值（原型前/中要定）

- 固定**行動上限**是多少（先前提到 10／6，兩數關係待確認；原型先取一個值試）。
- **移動 / 標記 / 揭露**各花幾點（＝各推時間軸多遠）。
- 控制範圍（視野）半徑 **N**；與「標記射程」是同一個數還是分開。
- 揭露到雷要不要**退行動值**（＝重現經典「續手」的爽感、獎勵正確推理）。
- 領先曝光的**成長曲線**（雷數 → 可見半徑怎麼長）。
- 本模式**盤面大小**（沿用 16×16？或為了移動改小場）。

---

## 6. 唯一真正的新工程量

迷霧（對手標記只在近處可見）代表伺服器**不能再對所有人廣播同一份狀態**：要**按玩家各自算可見範圍、送不同視圖**。

- 好消息：現有伺服器本就握有完整真相，只是多一層「依收件人過濾」。
- **Replays / 觀戰維持上帝視角**（看全圖反而更好看），只有即時對戰要迷霧。
- 可控，但這是本玩法唯一真正的新增工程。
